二、Spring是什么？能做什么？

  1. Spring是什么？（万能工具箱）

    解释①：用来装JavaBean的一种容器，即用来管理JavaBean实例化对象的一种容器。

    可以通过@bean自动完成对象的实例化。

    解释②：为了解决对象的创建和管理。

    解释③：Spring 让 Java 开发从“手工作坊”变成“现代化工厂”，专注业务逻辑，而不是重复造轮子。

  2. Spring能做什么？

    ①预定模板||依赖注入（如数据库操作只需要直接让Spring准备好即可，不需要去自己去写。）

    ②解耦、简化开发、易于测试；

    ③对事务进行声明；（事务是逻辑上的一组操作，要么都执行，要么都不执行）（如对数据库操作）等。

    如果把写代码比作造房子‌，传统方式需要自己搬砖、砌墙、接电线，而 Spring 提供了：

    • ‌预制板‌（依赖注入）：直接组装现成的模块。
    • ‌智能管家‌（AOP）：自动处理水电安装、刷墙等重复工作。
    • ‌装修模板‌（Spring Boot）：一键生成基础户型，省去画图纸的时间。
    • 用 ‌Spring Cloud‌ 可以快速构建分布式系统（比如电商中的订单、支付等独立服务），解决服务调用、负载均衡等问题。
    • 通过 ‌Spring MVC‌ 框架，可以轻松处理网页的请求、响应、路由等，让前后端协作更清晰。Spring Boot‌ 进一步简化配置，可以更快启动一个Web项目。
    • Spring 整合了 JDBC、JPA 等工具，让数据库操作（增删改查）变得更简单，减少繁琐代码。提供 ‌事务管理‌，保证数据库操作的一致性（比如转账失败时自动回滚，避免数据错误）。

一、基础概念

  1. 冯·诺依曼计算机

    组成：

    运算器、存储器、控制器（控制、指挥程序和数据的输入运行及处理结果）、输入设备、输出设备；

    特点：

    指令和数据以同等地位存放于存储器内，可按地址寻访；

    指令和数据均用二进制数表示；

    指令由操作码（操作）和地址码（操作数在存储器中的位置）组成；

    指令在存储器中是顺序存储的。

    机器以运算器为中心，输入输出设备与存储器间的数据传送通过运算器完成。

软件工程导论

发表于 2025-03-13 更新于 2025-03-23 分类于 软件工程

一. 软件危机

  1. 定义：软件危机是指在计算机软件的开发和维护过程中所遇到的一系列严重问题。
    表现：
    （1）开发成本和进度的估计不准确；（开发滞后）
    （2）用户对开发的软件不满意；
    （3）软件的质量靠不住；
    （4）软件不可维护；（重复开发相同的软件）
    （5）文档不完备；
    （6）软件开发成本在计算机系统总成本中所占的比例逐年上升；
    （7）软件开发生产率跟不上计算机应用的发展；（供不应求）
    等等。
  2. 产生原因：
    （1）软件本身的特点有关；（软件本身）
    （2）软件开发与维护的方法不正确；（软件开发不匹配计算机发展）
    （3）对用户要求没有完整准确的认识就匆忙着手编写程序；（开发人员）
  3. 消除方法：
    （1）对计算机软件有一个正确的认识；（消除软件就是程序的错误）
    软件定义：软件是程序，数据及相关文档的完整集合；
    （2）软件开发不是某个体劳动的神秘技巧，而是一种团队配合的工程项目，必须吸收和借鉴人类长期以来积累的原理、概念、技术和方法，尤其是教训；
    （3）推广实践中总结出来的经验、技术和方法，并研究新的；
    （4）开发和使用更好的软件工具。
  4. 总结：技术措施（方法、工具），必要的组织管理措施。
