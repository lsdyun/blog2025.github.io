DOM和BOM

一、DOM（Document Object Model ,文档对象模型）📄

‌定义‌：将 HTML/XML 文档解析为树形结构对象
‌作用‌：操作网页内容
‌核心对象‌：document
‌主要功能‌：

  1. ‌节点操作

  2. ‌内容修改

  3. ‌样式控制

  4. ‌事件处理

‌示例场景‌：动态更新列表、表单验证、页面元素交互

二、BOM（Browser Object Model，浏览器对象模型）🌐

‌定义‌：提供与浏览器窗口交互的对象模型
‌作用‌：控制浏览器行为
‌核心对象‌：window（可省略前缀）
‌主要组件‌：

对象 作用 示例
navigator 浏览器信息 navigator.userAgent
location URL操作 location.href = 'https://...'
history 浏览历史 history.back()
screen 屏幕尺寸 screen.width
localStorage 本地存储 localStorage.setItem('key', 'value')

‌常用方法‌：

‌示例场景‌：页面跳转、浏览器检测、本地数据存储

三、DOM vs BOM 对比表

特性 DOM BOM
‌标准化‌ W3C 标准 无官方标准，浏览器自行实现
‌操作对象‌ 网页文档 浏览器窗口
‌核心对象‌ document window
‌典型应用‌ 元素增删改查 页面跳转、弹窗、存储
‌跨浏览器‌ 一致性较高 部分API存在兼容问题

五、常见误区

  1. ‌**❌ window 是BOM的全部**‌ → 还包含location/navigator等对象
  2. ‌**❌ DOM只能操作现有元素**‌ → 可动态创建新元素
  3. ‌**❌ BOM完全标准化**‌ → 不同浏览器可能实现不同（如IE的attachEvent vs 标准的addEventListener）

‌总结‌：

  • ‌DOM‌ 是你的网页手术刀（精确操作内容）
  • ‌BOM‌ 是你的浏览器遥控器（控制浏览器行为）
    两者配合使用，才能实现完整的网页交互功能。
