生产环境

  • 自动使用 .env.production 配置

配置说明

文件/配置项 作用说明 修改场景
.env.development 开发环境配置 修改本地开发后端地址
.env.production 生产环境配置 部署时修改线上地址
vue.config.js 中的 proxy 开发环境代理配置 需要更改代理规则时修改
request.js 中的拦截器 统一处理认证/错误/响应 需要调整全局逻辑时修改

优势说明

  1. ‌环境隔离‌

    • 开发环境使用代理解决跨域
    • 生产环境直连正式 API 地址

  2. ‌零代码修改部署‌
    只需修改 .env.production 文件中的 VUE_APP_API_BASE 值即可切换生产环境地址

  3. ‌统一请求管理‌

    • 自动携带 Token
    • 全局错误处理
    • 统一响应格式处理

项目上线

上线部署后，浏览器的访问的 url （包括端口）是部署工具自动分配的，不需要也不能后续去改（因为存在审核的问题），后端的 api 可以通过 .env.production 去修改。

‌一、前端 URL 的确定性‌

  1. ‌自动分配机制‌
    前端上线后的访问 URL（含端口）由部署服务器配置决定，例如：

    • 若需修改访问端口或域名，需调整服务器配置而非前端代码
    • 前端代码中无需、也无法硬编码端口信息

  2. ‌审核相关限制‌
    若部署流程涉及运维审核（如企业发布流程），修改服务器配置可能需要审批，但本质上仍属于技术可调整范畴

‌二、后端 API 的可控性‌

  1. ‌环境变量动态管理‌
    后端接口地址通过 .env.production 中的 VUE_APP_API_BASE 配置：

    • 修改后需重新构建部署前端项目以生效

  2. ‌无需后端服务调整‌
    后端服务本身只需保持接口路径不变，前端通过环境变量指向新地址即可实现切换

总结：

✅ 前端 URL 由部署工具分配，修改需调整服务器配置
✅ 后端 API 地址通过环境变量灵活管理
