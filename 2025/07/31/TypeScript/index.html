TypeScript

简介：

定义：

TypeScript 是基于 JavaScript 语言的编程语言。目的是增强 JavaScript 的功能。

TypeScript 可以看成是 JavaScript 的超集（superset），即它继承了后者的全部语法，

TypeScript 在 JavaScript 的基础上添加了一个独立的类型（type）系统。

TypeScript 设计目标是开发大型应用。

TypeScript 支持 ES6 标准。

TypeScript 是一种面向对象的编程语言。

运行逻辑：TypeScript -> Javascript（编译）

TypeScript 脚本文件使用.ts后缀名，JavaScript 脚本文件使用.js后缀名。

tsc 编译器的作用就是把.ts脚本转变成.js脚本。（tsc命令）

tsconfig.json是 TypeScript 项目的配置文件，放在项目的根目录。

特性：

类型：（严格）

类型（type）指的是一组具有相同特征的值。如果两个值具有某种共同的特征，就可以说，它们属于同一种类型。

TypeScript 的类型其实更像是 Java 里的变量类型（int、String等），严格限制定义的变量必须是对应类型的值。如：（静态性）

JavaScript 的类型则没有这个限制，如：（动态性）

TypeScript 引入静态类型这个东西，目的为：

  • 就是为了在编码阶段去提前发现和解决问题。（节省程序员时间）

  • 有利于代码的静态分析。（不必运行代码就可确定变量类型，这点在大型项目尤为重要）

  • 方便 集成开发环境 IDE 做到自动补全和语法提示。

  • 方便程序员直接观看代码确定代码作用。

  • 方便代码重构

TypeScript 引入静态类型的缺点：

  • 丧失 JavaScript 的灵活性（类型定义）
  • 增加编程工作量
  • 学习成本更高
  • 引入了“单独校验类型正确”的编译步骤，而 JavaScript 直接就可在引擎上运行
  • 兼容性问题（过去的 JavaScript 代码）

接口：Interface

接口可以定义属性和方法，还可以通过 implements 关键字实现接口，或者通过 extends 进行扩展，便于定义复杂的数据类型。

枚举：Enum

元组：Tuple

元组允许定义具有固定数量和类型的数组。它适用于需要固定数据结构的场景，比如坐标或 RGB 颜色值。

访问控制修饰符 (Access Modifiers)

TypeScript 在类中提供了 public、private 和 protected 修饰符，允许控制属性或方法的可见性，支持更好的封装。

抽象类 (Abstract Classes)

TypeScript 支持抽象类，抽象类不能直接实例化，需要由子类实现。抽象类适用于定义通用行为和抽象方法的类层次结构。

泛型 (Generics)

TypeScript 支持泛型，允许在类、接口和函数中使用参数化类型，使得代码可以适应不同的类型需求，同时保持类型安全。

模块和命名空间：import 和 export

TypeScript 提供了基于 ES6 的模块系统，使用 import 和 export 导入和导出模块。此外，TypeScript 还支持命名空间（Namespace），用于组织代码和避免命名冲突。

类型守卫 (Type Guards)

TypeScript 提供了类型守卫，可以在代码中检查变量类型，帮助编译器推断更加具体的类型。这对于联合类型尤为重要。

可选链和空值合并运算符

TypeScript 增加了 JavaScript 的可选链 (?.) 和空值合并运算符 (??)，简化了代码中对可能为 null 或 undefined 值的处理。

  • any：“顶层类型”（top type）

    表示没有任何限制，该类型的变量可以赋予任意类型的值。

    应用场景：

    • 出于特殊原因，需要关闭某些变量的类型检查，就可以把该变量的类型设为any。
    • 为了适配以前老的 JavaScript 项目，让代码快速迁移到 TypeScript，可以把变量类型设为any。

    问题：

    • 建议使用let和var声明变量时，如果不赋值，就一定要显式声明类型，否则可能存在安全隐患。

    • any类型除了关闭类型检查，还有一个很大的问题，就是它会“污染”其他变量。它可以赋值给其他任何类型的变量（因为没有类型检查），导致其他变量出错。

  • unknow：“顶层类型”（top type）”

    为了解决any类型“污染”其他变量的问题，TypeScript 3.0 引入了unknown类型，不像any那样自由，可以视为严格版的any。

    unknown类型跟any类型的不同之处在于，它不能直接使用，有以下几个限制：

    • unknown类型的变量，不能直接赋值给（除了any类型和unknown类型之外的）其他类型的变量。

    • 不能直接调用unknown类型变量的方法和属性。

      • 只有经过“类型缩小”，unknown类型变量才可以使用。

      • unknown类型变量能够进行的运算是有限的，只能进行比较运算（运算符==、===、!=、!==、||、&&、?）、取反运算（运算符!）、typeof运算符和instanceof运算符这几种，其他运算都会报错。

    • 凡是需要设为any类型的地方，通常都应该优先考虑设为unknown类型。

  • never：“底层类型（bottom type）”

    • 特点：

      可以赋值给任意其他类型。（空集是任何集合的子集。）

      never类型是任何其他类型所共有的

装饰器（Decorator）

装饰器（Decorator）是一种语法结构，用来在定义时修改类（class）的行为。

装饰器一般只用来为类添加某种特定行为。

（1）第一个字符（或者说前缀）是@，后面是一个表达式。

（2）@后面的表达式，必须是一个函数（或者执行后可以得到一个函数）。

（3）这个函数接受所修饰对象的一些相关值作为参数。

（4）这个函数要么不返回值，要么返回一个新对象取代所修饰的目标对象。

declare 关键字：

declare 关键字用来告诉编译器，某个类型是存在的，可以在当前文件中使用。

一般用于当前文件引入外部库定义的函数时，不知道外部库函数的类型，就可以在当前文件使用declare 关键字告知编译器外部库函数的类型。
