一、定义

Hibernate 是 Java 生态中广泛使用的 ‌ORM（对象关系映射）框架‌，用于简化数据库操作，将 Java 对象与关系型数据库表自动映射，提供高效、面向对象的数据访问能力。

结构：

二、特点

  1. ‌ORM 映射-‌->O代表的是Objcet，R代表的是Relative，M代表的是Mapping
    • 自动将 Java 类映射到数据库表，属性映射到字段，减少手动 SQL 编写。
    • 支持继承、多态等面向对象特性在关系型数据库中的实现。
  2. ‌数据库无关性‌
    • 通过 Hibernate Dialect 适配不同数据库（如 MySQL、Oracle），切换数据库只需修改配置。
  3. ‌高效性能‌
    • 提供一级缓存（Session 级别）、二级缓存（SessionFactory 级别）和查询缓存，减少数据库访问频率。
    • 延迟加载（Lazy Loading）机制优化资源使用。
  4. ‌简化事务管理‌
    • 与 Spring 框架无缝集成，支持声明式事务（@Transactional）。

三、组件

  1. ‌配置文件‌
    • ‌**hibernate.cfg.xml**‌：全局配置数据库连接、缓存策略、Dialect 方言（不同于其他数据库的语言）等。
    • ‌映射文件（\*.hbm.xml）或注解‌：定义对象与表的映射关系。
  2. ‌核心接口与类‌
    • ‌**SessionFactory**‌：重量级对象，管理数据库连接池和二级缓存，全局唯一。
    • ‌**Session**‌：轻量级对象，代表一次数据库会话，提供 CRUD 方法（如 save(), get(), update()）。
    • ‌**Transaction**‌：管理事务的提交与回滚。
    • ‌**Query / Criteria**‌：执行 HQL（Hibernate Query Language）或 Criteria 查询。

五、‌Hibernate和JDBC区别

特性‌ ‌Hibernate‌ ‌JDBC‌
‌抽象级别‌ 面向对象（ORM） 面向 SQL（低层 API）
‌开发效率‌ 高（自动映射、代码生成） 低（手动编写 SQL 和映射）
‌性能‌ 需合理配置缓存和查询 直接控制，理论上更高效
‌适用场景‌ 复杂业务对象模型、快速开发 简单操作、高性能要求场景

六、Hibernate和MyBatis区别

组件/文件‌ ‌Hibernate‌ ‌MyBatis‌
‌实体类‌ 使用 JPA 注解（如 @Entity, @Table）或 XML 映射文件定义对象与表的映射关系，需包含关联关系配置（如 @OneToMany）‌ 实体类通常为 POJO，无需复杂注解（仅需字段与表字段对应），关联关系通过 SQL 或 ResultMap 手动处理‌
‌配置文件‌ 全局配置文件 hibernate.cfg.xml，管理数据库连接、缓存、Dialect 等‌ 全局配置文件 mybatis-config.xml，定义数据源、插件及全局设置，每个 Mapper 接口需对应 XML 文件或注解 SQL‌
‌数据访问层‌ DAO 类通过 Session 操作数据库（如 session.save()），方法中无需显式 SQL‌ Mapper 接口与 XML 文件（或注解）绑定，需手动编写 SQL 语句及结果映射（如 <select id="...">）‌
‌工具类‌ 需 SessionFactory 管理会话（如 HibernateUtil）‌ 需 SqlSessionFactory 管理会话（如 MyBatisUtil），通常与 Spring 集成后自动管理‌

  • ‌Hibernate‌

    • ‌自动 SQL 生成‌：通过实体类注解自动生成 DDL/DML，减少手动 SQL 编写‌。
    • ‌关联关系管理‌：支持级联操作（如 CascadeType.ALL）和延迟加载（FetchType.LAZY）‌。
    • ‌事务整合‌：通常与 Spring 的 @Transactional 注解结合使用‌。

  • ‌MyBatis‌

    • ‌SQL 灵活性‌：需手动编写 SQL，支持动态 SQL（如 <if>, <foreach>）及存储过程调用‌。
    • ‌结果集映射‌：通过 <resultMap> 或注解显式定义字段与对象属性的映射关系‌。
    • ‌插件扩展‌：支持自定义插件（如分页插件 PageHelper）优化 SQL 执行流程‌。

  • ‌Hibernate‌

    • ‌适用场景‌：领域模型复杂、需快速开发且对 SQL 优化要求较低的项目（如企业内部系统）‌。
    • ‌结构特点‌：代码简洁，ORM 配置集中，但可能因自动生成 SQL 导致性能优化困难‌。

  • ‌MyBatis‌

    • ‌适用场景‌：需高度 SQL 控制、复杂查询优化或遗留数据库迁移的项目（如互联网高并发系统）‌。
    • ‌结构特点‌：SQL 分散在 XML 或注解中，灵活性高但维护成本增加‌。
