一、Spring Security概览

  • Spring Security中文文档

  • Spring Security + JWT + MySQL 实现基于 Token 的身份认证

  • 教程

  • 兼容性：

    • Spring Boot 3—Spring Security 6.2.x
    • Spring Boot 2—Spring Security 5.8.x

  • Spring Security 是一个提供身份认证、授权和防护常见攻击的框架。

    • 身份认证（Authentication）：

      • 验证用户身份（如用户名/密码、短信验证等），解决“你是谁”的问题‌；
      • 通过UserDetailsService接口实现自定义用户数据源查询‌；
      • 支持表单登录、OAuth2、LDAP等多种认证方式，默认提供基础表单登录页‌；
      • 密码加密存储需使用PasswordEncoder接口实现类（如BCrypt算法）‌；

    • 权限控制（Authorization）

      • 根据用户权限控制资源访问（如角色分配、URL权限控制），解决“你能做什么”的问题‌
      • 基于角色（Role）或权限（Permission）的访问控制‌；
      • URL级别‌：通过配置HttpSecurity限制访问路径‌；
      • 方法级别‌：使用@PreAuthorize、@Secured等注解实现细粒度控制‌
      • 常用注解：@PreAuthorize、@Secured、@PostAuthorize‘

    • 安全防护机制

      • ‌CSRF防护：默认启用，需配合模板引擎或手动配置令牌‌；

      • 会话管理‌：防止会话固定攻击，支持并发会话控制‌；

      • HTTP安全头：自动配置X-Content-Type-Options等头部增强防护‌；

  • Spring Security集成与扩展

    • 与Spring Boot整合
      • 通过spring-boot-starter-security快速启用默认安全配置‌
      • 自定义安全配置需继承WebSecurityConfigurerAdapter类‌
    • 第三方认证支持
      • OAuth2‌：实现单点登录（SSO）和第三方授权（如微信、GitHub）‌
      • ‌JWT‌：结合Token实现无状态REST API认证‌
        • Token/JWT令牌：用于通信双方之间传递安全信息的简洁的、URL安全的字符串，由三部分组成，头部、载荷与签名
    • 数据库集成
      • 通过MyBatis/JPA实现用户角色数据持久化‌
      • 需设计用户表、角色表及关联表，配合UserDetailsService加载权限‌

二、Spring Security身份认证

  1. 流程：

    Spring Security 的认证流程由过滤器链驱动，核心步骤如下：

    • 用户提交登录请求，触发 UsernamePasswordAuthenticationFilter 拦截请求‌
    • 通过 AuthenticationManager 委托 AuthenticationProvider 进行认证（如 DaoAuthenticationProvider）‌
    • UserDetailsService 加载用户信息（用户名、密码、权限）并与输入信息比对‌
    • 认证成功后生成 Authentication 对象，存入安全上下文（SecurityContextHolder）‌

    核心组件：

    • ‌**UsernamePasswordAuthenticationFilter**‌：处理表单登录请求，默认拦截 /login POST 请求‌
    • ‌**UserDetailsService**‌：定义用户数据加载接口，需开发者自行实现（如从数据库读取）‌
    • ‌**PasswordEncoder**‌：密码加密策略（推荐使用 BCryptPasswordEncoder）‌

    • Authentication接口：它的实现类，表示当前访问系统的用户，封装了用户相关信息。

      • SeucirtyContext：上下文对象，用来获取Authentication
      • SecurityContextHolder：上下文管理对象，用来在程序任何地方获取SecurityContext
      • Principal：用户信息，没有认证时一般是用户名，认证后一般是用户对象
      • Credentials：用户凭证，一般是密码
      • Authorities：用户权限

    • AuthenticationManager接口：定义了认证Authentication的方法。

      • 逻辑：根据用户名先查询出用户对象(没有查到则抛出异常)将用户对象的密码和传递过来的密码进行校验，密码不匹配则抛出异常。
      • authenticate()：执行身份验证

    • UserDetailsService接口：加载用户特定数据的核心接口，内含一个根据用户名查询用户信息的方法。

      • loadUserByUsername(String username)：唯一方法，目的通过用户名查询用户对象。

    • UserDetails接口：提供核心用户信息（账号、密码）。

权限控制实现方式：

  1. URL 级权限控制

    使用 HttpSecurity 配置 URL 匹配规则，支持多种表达式：

    • ‌IP 限制‌：.antMatchers("/**").hasIpAddress("192.168.1.0/24")‌
    • ‌混合条件‌：.access("hasRole('ADMIN') and hasIpAddress('192.168.1.10')")‌
    • ‌动态表达式‌：结合自定义逻辑实现动态权限判断‌

  2. 方法级权限控制

    通过注解实现精确到方法级别的控制（需启用 @EnableMethodSecurity）

    • ‌**@PreAuthorize**‌：方法执行前校验权限

    • **@PostAuthorize**‌：方法执行后校验返回值权限

    • **@Secured**‌：简化角色校验（需启用 securedEnabled = true）

  3. ‌动态权限控制

    结合数据库动态加载权限规则，典型实现步骤：

    • 实现 FilterInvocationSecurityMetadataSource 接口加载权限-资源映射；
    • 自定义投票器（AccessDecisionVoter）实现动态权限决策‌。

四、Spring Security安全防护机制

核心防护机制:

  1. CSRF（跨站请求伪造）防护

    Spring Security 默认启用 CSRF 防护，防止恶意网站利用用户已登录状态发起非法请求‌

    • 实现原理‌：为表单请求生成唯一 _csrf 令牌，服务端验证请求是否携带有效令牌‌

    • ‌配置示例‌（禁用场景）

  2. 会话管理

    提供会话固定攻击防护（sessionFixation().migrateSession()）及并发会话控制‌。

    • ‌基于 Token 的无状态会话‌：适用于前后端分离架构，如 JWT 方案‌。
    • ‌会话超时与失效‌：支持自定义会话失效时间与失效跳转路径‌。

  3. 密码加密机制

    强制使用安全算法（如 BCryptPasswordEncoder）加密用户密码，避免明文存储‌

  4. CORS（跨域资源共享）处理

    通过 CorsConfigurationSource 配置允许跨域的域名、请求方法及头部信息‌

  @Configuration
  @EnableWebSecurity
  public class SecurityConfig {
   @Bean
   public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
       http
           .csrf().disable()
           .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS) // 禁用 Session
           .and()
           .authorizeRequests()
           .antMatchers("/login").permitAll()
           .anyRequest().authenticated()
           .and()
           .addFilterBefore(new JwtAuthenticationFilter(), UsernamePasswordAuthenticationFilter.class);
       return http.build();
   }
  }

#### **注意事项**

- **无状态设计**：使用 `SessionCreationPolicy.STATELESS` 禁用 Session，完全依赖 JWT。
- **安全性**：JWT 应通过 HTTPS 传输，密钥需妥善保管。
- **Token 刷新**：可通过 Refresh Token 机制延长用户会话。
- **黑名单**：如需实现即时注销，需结合 Redis 维护 Token 黑名单。

通过上述流程，Spring Security 与 JWT 实现了无状态、可扩展的认证鉴权机制。
