一、MyBatis-plus初级

官方网站

  1. MyBatis-plus新功能

    • 增强版的MyBatis，简化开发，提高效率；

    • 启动会自动注入CRUD，性能无损耗；

    • 内置通用Mapper，通用Service；

    • 支持Lambda形式调用；

    • 支持主键自动生成；

    • 内置代码生成器，可快速生成Mapper、Model、Service、Controller代码；

    • 内置分页插件（MyBatis物理分页-LIMIT），支持多种数据库；

    • 内置性能分析插件；

    • 内置全局拦截插件；

  5. 注解

    • @MapperScan("com.example.mapper")

      扫描指定包下的Mapper接口，避免在每个Mapper接口上单独添加@Mapper注解。

    • @ComponentScan

      组件扫描扩展，自定义Spring组件扫描范围，但不直接关联MyBatis Mapper扫描。

    • @TableName("数据库表名")

      在实体类中，用于表名注解，标识实体类对应的表

    • @TableId(value="id",type="IdType.AUTO")

      在实体类中，用于主键字段，IdType.AUTO自增

    • @TableField("数据库表中属性名")

      @TableField(value="name",fill=FieldFill.INSERT)

      fill：自动填充策略（插入/更新时自动填充值）

      • DEFAULT：不处理
      • INSERT：插入时填充
      • UPDATE：更新时填充
      • INSERT_UPDATE：插入和更新时填充

      在实体类中，用于字段注解上方

    • @EnumValue

      普通枚举类注解的枚举字段上

    • @OrderBy

      内置 SQL 默认指定排序，优先级低于 wrapper 条件查询

    • @TableLogic

      表字段逻辑处理注解（逻辑删除）

    • @KeySequence

      序列主键

    • @Version

      乐观锁注解字段，（更新时自动检查版本号）

      乐观锁：总是假设最好的情况，每次去拿数据的时候都认为别人不会修改，所以不会上锁，只在更新的时候会判断一下在此期间别人有没有去更新这个数据。

      乐观锁和悲观锁

    • @SqlParser

      SQL解析过滤

    • @InterceptorIgnore

      拦截器忽略，动态忽略 MyBatis-Plus 内置拦截器（如分页、多租户）

    • @AutoResultMap

      自动生成结果映射，配合 @TableName(autoResultMap = true) 自动生成复杂映射

  2. CRUD接口

    通过继承 BaseMapper 获得通用 CRUD 方法，不需要写SQL代码

    • 查

      方法名 说明 示例
      T selectById(Serializable id) 根据 ID 查询 userMapper.selectById(1L)
      List<T> selectBatchIds(Collection<?> idList) 批量查询（根据 ID 列表） userMapper.selectBatchIds(Arrays.asList(1,2,3))
      List<T> selectByMap(Map<String, Object> columnMap) 根据字段名-值的 Map 条件查询‌ Map.of("status", 1) → 查询 status=1 的记录
      List<T> selectList(Wrapper<T> wrapper) 根据 Wrapper 条件查询列表‌ 结合 QueryWrapper 动态构建条件‌
      T selectOne(Wrapper<T> wrapper) 查询单条记录（需确保结果唯一）‌ 动态条件查询唯一用户
      Long selectCount(Wrapper<T> wrapper) 统计符合条件的数据总数‌ 统计 age > 25 的用户数量‌

    • 增加

      方法名 说明 示例
      int insert(T entity) 插入一条记录 userMapper.insert(user)
      int insertBatch(Collection<T> list) 批量插入（需数据库驱动支持批量操作） userMapper.insertBatch(users)

    • 修改

      方法名 说明 示例
      int updateById(T entity) 根据 ID 更新记录 userMapper.updateById(user)
      int update(T entity, Wrapper<T> wrapper) 根据条件更新记录 userMapper.update(user, new UpdateWrapper<User>().eq("name", "Alice"))
      int updateBatchById(Collection<T> entityList); 根据ID 批量更新

    • 删除

      方法名 说明 示例
      int deleteById(Serializable id) 根据 ID 删除 userMapper.deleteById(1L)
      int deleteBatchIds(Collection<?> idList) 批量删除（根据 ID 列表） userMapper.deleteBatchIds(Arrays.asList(1,2,3))
      int deleteByMap(Map<String, Object> columnMap) 根据字段名-值的 Map 条件删除‌ Map.of("age", 20) → 删除 age=20 的记录
      int delete(Wrapper<T> wrapper) 根据条件构造器 Wrapper 删除‌ 结合 QueryWrapper 构建动态条件‌

    • 更多接口

      • Save
      • Remove
      • Get

      • Count
